## Почему лучше использовать `IS DISTINCT FROM` в триггерных функциях?

Это важно, чтобы правильно обрабатывать `NULL` значения. Такой способ проверки позволяет избежать проблем, связанных с традиционными операторами сравнения. Если сравнивать через обычное равно (или не равно), можно неожиданно получить `NULL` при сравнении с `NULL`:

```sql
SELECT 1 <> NULL a
     , 2 = NULL b
     , NULL = NULL c
     , NULL <> NULL d;
```

Результат:

| a      | b      | c      | d      |
| ------ | ------ | ------ | ------ |
| `NULL` | `NULL` | `NULL` | `NULL` |

А если использовать `IS DISTINCT FROM`, то вы получите то, что ожидаете:

```sql
SELECT 3 IS DISTINCT FROM NULL e
     , 4 IS NOT DISTINCT FROM NULL f
     , NULL IS DISTINCT FROM NULL g
     , NULL IS NOT DISTINCT FROM NULL h;
```

Результат:

| e      | f       | g       | h      |
| ------ | ------- | ------- | ------ |
| `TRUE` | `FALSE` | `FALSE` | `TRUE` |


Внутри триггерной функции при сравнении старого и нового значений лучше использовать `IS DISTINCT FROM`, чтобы получить правильную детекцию изменений. Это обеспечит надежность логики обработки.
